*MVC 모델 : 웹뿐만 아니라 다양한 개발에 적용된다
	- 사용자 인터페이스, 데이터 및 논리 제어를 구현하는데 사용되는 소프트웨어 디자인 패턴
	- 소프트웨어의 비즈니스 로직과 화면을 구분하는데 중점을 둔다 (관심사 분리)

M - Model : 비즈니스 로직 - 기능(로그인 기능, 회원가입 기능...)
					서비스(기능) - Service
						- DTO(Data Transfer Object) : 데이터 전달용 객체
							- 멤버 변수 + getter, setter
						- DAO(Data Access Object) : 데이터 접근용 객체						
						- 기타(다른 서비스 객체, 검증용 객체(Validator))
						- 객체간의 협력을 통해서 기능을 완성한다
						- 객체간의 의존성
						
					
V - View : 요청 처리 결과를 출력(jsp)

C - Controller : 요청과 응답 사이의 중재자
	요청		 		서블릿(servlet)								응답
					요청 처리 위한 서비스(모델)
